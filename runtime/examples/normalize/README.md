# KDSE Normalization Examples

This directory contains examples of KDSE Normalization in action.

---

## Example 1: Simple Repository

### Before Normalization

```
my-simple-repo/
├── README.md
├── LICENSE
└── docs/
    └── api.md
```

### Run Normalization

```bash
cd my-simple-repo
kdse normalize
```

### After Normalization

```
my-simple-repo/
├── README.md              ← Original (unchanged)
├── LICENSE               ← Original (unchanged)
├── docs/
│   └── api.md            ← Original (unchanged)
└── .kdse/
    └── normalized/
        ├── SPEC.md         ← KDSE Specification
        ├── ARCHITECTURE.md ← (generated if content warrants)
        ├── GLOSSARY.md     ← (generated if terms found)
        ├── GOVERNANCE.md   ← Governance info from LICENSE
        └── ARTIFACTS.md    ← Artifact index
```

## Example 2: Architecture-Heavy Repository

### Before Normalization

```
my-architecture-repo/
├── README.md
├── docs/
│   ├── getting-started.md
│   └── guide.md
├── architecture/
│   ├── overview.md
│   └── decisions.md
├── adr/
│   ├── ADR-001-use-postgres.md
│   └── ADR-002-rest-api.md
└── SPEC.md
```

### Run Normalization

```bash
cd my-architecture-repo
kdse normalize
```

### After Normalization

```
my-architecture-repo/
├── README.md              ← Original (unchanged)
├── docs/                  ← Original (unchanged)
├── architecture/          ← Original (unchanged)
├── adr/                   ← Original (unchanged)
├── SPEC.md                ← Original (unchanged)
└── .kdse/
    └── normalized/
        ├── SPEC.md          ← KDSE Specification (from original SPEC.md)
        ├── ARCHITECTURE.md  ← KDSE Architecture (from architecture/)
        ├── GLOSSARY.md      ← Extracted terms
        ├── DECISIONS.md     ← Architecture Decisions (from ADR files)
        ├── GOVERNANCE.md    ← Governance info
        └── ARTIFACTS.md     ← Artifact index
```

## Example 4: Generated DECISIONS.md

When ADRs are found, `kdse normalize` generates a consolidated decisions document:

```markdown
# Architecture Decisions

**Document Version:** 1.0  
**Type:** Engineering Artifact  
**Phase:** Derived  

---

## Decisions

### ADR-001: Use PostgreSQL

**Status:** Accepted

**Context:**

We need a database that supports relational data, transactions, and complex queries.

**Decision:**

Use PostgreSQL as the primary database.

**Rationale:**

PostgreSQL provides ACID compliance, excellent performance, and rich feature set.

*Source: adr/ADR-001-use-postgres.md*

---

### ADR-002: REST API

**Status:** Accepted

**Context:**

We need a standard way to expose services to clients.

**Decision:**

Use RESTful API design with JSON payloads.

*Source: adr/ADR-002-rest-api.md*

---

*This decisions document was generated by KDSE Normalization from existing documentation.*
```

## Workflow Examples

### Workflow A: Immediate Start

```bash
# No normalization needed
kdse run
# Start working with the current repository
```

### Workflow B: Normalize First

```bash
# Normalize existing docs
kdse normalize

# Then run with normalized context
kdse run
```

### Workflow C: Normalize Later

```bash
# Start working immediately
kdse run

# After some work, normalize
kdse normalize

# Continue with both views
kdse run
```

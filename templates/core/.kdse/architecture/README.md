# Architecture Directory

This directory contains system design artifacts.

## Purpose

The architecture directory stores design documents that define:
- System boundaries and context
- Component structure
- Data models
- Integration patterns

## Structure

- `001-system-context.md` - System boundaries and external interactions
- `002-components.md` - Component breakdown and responsibilities
- `003-data-model.md` - Data structures and relationships

## Usage

Architecture artifacts should be:
- Derived from knowledge entries
- Updated when requirements change
- Verified against implementation

## Guidelines

- Use evidence to justify design decisions
- Reference knowledge entries where applicable
- Include diagrams where helpful
- Track architectural decisions with rationale

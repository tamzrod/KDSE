# Documentation Directory

This directory contains project documentation and user guides.

## Purpose

The docs directory stores:
- User documentation
- Developer guides
- API documentation
- Runbooks

## Structure

```
docs/
├── user/
│   ├── getting-started.md
│   └── user-guide.md
├── developer/
│   ├── setup.md
│   ├── architecture.md
│   └── api.md
└── runbooks/
    ├── deployment.md
    └── troubleshooting.md
```

## Usage

Documentation should be:
- Kept up-to-date with code
- Written for the target audience
- Searchable and well-organized
- Linked from relevant artifacts

## Guidelines

- Use markdown format
- Include code examples
- Add screenshots where helpful
- Review and update regularly
